Axxanoid Marketplace: Developer Documentation

1. Architectural Overview & Standards

The Axxanoid Marketplace plugin relies on strict separation of concerns. PHP controllers handle logic and routing exclusively. All structural HTML is housed in /templates/template-parts/, and all styling is enqueued via external .css files. The system is designed to interface with native WooCommerce taxonomies (Brands and Categories) to prevent data siloing.

2. Data Schema & Relationships

The system relies on a 1-to-1-to-Many relationship across WordPress and WooCommerce.

  • The Maker (axx_market_maker CPT): Acts as the administrative and subscription hub. Stores the Maker's email, trial expiration date, and active WooCommerce Subscription Order ID.
  • The Brand (WooCommerce Taxonomy): Each Maker CPT is mapped 1:1 with a native WooCommerce Brand ID.
  • The Products (WooCommerce product): Scraped items are created as external WooCommerce products. They are assigned to the Maker's Brand ID and pushed into the "Indie Finds" parent category.

3. REST API Endpoints

The system opens specific endpoints for the headless Python drones:

  • /wp-json/wp/v2/axx_market_maker: Accepts initial Maker payloads (Email, Shop URL) and initializes the 10-day Trial status.
  • /wp-json/wc/v3/products: Native Woo API used to inject the external Etsy products and assign them to the mapped Brand ID.
  • /wp-json/axx_market/v1/pending-pitches: Custom endpoint returning a clean JSON array of unpitched Makers for the Qwen 14B drone (Day 1).
  • /wp-json/axx_market/v1/pending-followups: Custom endpoint returning Makers who need their Day-5 "Pennies a day" reminder.
  • /wp-json/axx_market/v1/expiring-trials: Custom endpoint returning Makers who have hit Day 10 without an active subscription for Executioner processing.

4. The Category Mapper & Retroactive Sync

Because Etsy categories do not cleanly align with native WooCommerce categories, the plugin utilizes a Vue.js-powered Mapping UI in the Admin Dashboard.

  • The Rule Engine: Maps scraped tags (e.g., 3d-printed-stash) to native WooCommerce Category IDs.
  • The "Run Now" Feature: Allows administrators to trigger a retroactive AJAX loop. This function scans all existing WooCommerce products within the "Indie Finds" parent category and applies updated category mappings based on their scraped tags.

5. The Dunning, "Digital Rent" & SEO Harvest Engine

Unlike standard dropshipping, this plugin charges Makers "digital rent" via a manual, non-auto-renewing $2.99/mo invoice to eliminate B2B conversion friction.

  • Makers are given a 10-Day Free Trial upon ingestion.
  • Purchasing the WooCommerce "Marketplace Rent" product updates their marketplace_status to Active.
  • The SEO Harvest: If the trial expires unpaid on Day 10, the Executioner Drone changes the Maker CPT to draft. It then reassigns all of their WooCommerce products to a generic "Affiliate Finds" Brand, preserving the SEO value for Average Stoner.
  • The Pruner: At Day 45, unpaid products in the Affiliate brand are drafted to keep the catalog fresh.
